# Web Application Security Testing using DVWA

## 📌 Project Overview
This project demonstrates web application security testing using Damn Vulnerable Web Application (DVWA). The objective is to identify common web vulnerabilities, understand their impact, and learn basic mitigation techniques.

---

## 🎯 Objectives
- Perform security testing on DVWA.
- Learn common web vulnerabilities.
- Demonstrate the use of security testing tools.
- Document findings and recommendations.

---

## 🛠 Tools Used
- Kali Linux
- DVWA (Damn Vulnerable Web Application)
- Burp Suite Community Edition
- Firefox Browser
- MariaDB
- Apache2
- SecurityHeaders.com

---

## 🔍 Vulnerabilities Tested
- SQL Injection
- Reflected Cross-Site Scripting (XSS)
- Stored Cross-Site Scripting (XSS)
- Cross-Site Request Forgery (CSRF)
- Local File Inclusion (LFI)
- Burp Suite Intruder
- Security Headers Analysis

## 🛡 Security Recommendations
- Use parameterized queries to prevent SQL Injection.
- Validate and sanitize user input.
- Implement Content Security Policy (CSP).
- Enable CSRF protection.
- Restrict file inclusion.
- Configure proper HTTP Security Headers.
- Perform regular vulnerability assessments.

---

## 📚 Learning Outcomes
- Understood common web vulnerabilities.
- Learned to exploit vulnerabilities in a controlled lab environment.
- Used Burp Suite for intercepting and analyzing HTTP requests.
- Improved understanding of secure web application development.
